description: Cuándo commitear — el usuario avisa, no preguntar repetidamente
metadata:
  type: feedback
---

No preguntar "¿commiteo?" al final de cada cambio. El usuario avisa explícitamente cuando una etapa está finalizada y lista para commitear.

**Why:** Preguntar todo el tiempo molesta y corta el flujo de trabajo iterativo.
**How to apply:** Seguir implementando cambios sin ofrecer commit en cada turno. Commitear solo cuando el usuario lo pide o dice que la etapa terminó. Ver [[project-decisions]].
